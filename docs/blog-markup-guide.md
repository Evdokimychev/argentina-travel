# Как размечать статью блога

Короткая памятка для редакторов. Проза остаётся в `body`; структурные блоки можно задать явно или оставить авто-парсер.

## Авто-парсер (`body`)

Разделите блоки **двойным переносом строки** (`\n\n`).

| Формат | Результат |
|--------|-----------|
| `**Совет:** текст` | Callout «Совет» |
| `**Важно:**`, `**Лайфхак:**`, `**Внимание:**` | Callout с соответствующим типом |
| `**Бюджет:**`, `**Сезон:**` | Callout «Что нужно знать» |
| `**Ошибка туристов:**` | Callout ошибки |
| `> [!tip] Заголовок\nТекст` | Markdown-callout |
| `* пункт` или `- пункт` | Маркированный список |
| `1. шаг` | Нумерованные шаги |
| `□ пункт` | Чек-лист |
| `❌ пункт` | Чек-лист (негатив) |
| Таблица с `\t` | Таблица |
| Короткая строка без точки | Подзаголовок |

Секции с заголовком **FAQ** / **Частые вопросы** → аккордеон FAQ (`Вопрос? Ответ.`).

Секции **Типичные ошибки** → пары «заголовок + абзац» как callout ошибки.

## Typed blocks (CMS-ready)

В `BlogPostSection` опционально:

- `blockType` — `faq`, `mistakes`, `checklist` (подсказка парсеру)
- `blocks` — массив typed-блоков (рендерятся **после** parsed `body`)

Типы блоков: `map`, `ticket-link`, `seasons`, `budget`, `faq`, `table`, callout, checklist, steps.

Пример data-only (без правки прозы): `src/data/blog-typed-blocks/` — пилот `best-time-to-visit-argentina`.

## Rich-гиды

Полные typed-блоки в `src/data/blog-articles/*.ts` — map, ticket-link, seasons, gallery и др.
