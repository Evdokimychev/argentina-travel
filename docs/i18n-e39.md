# E39: i18n foundation (es/en)

## Scope (E39)

Pilot internationalization for **interface chrome** — navigation, footer, auth labels, booking CTAs, homepage hero.

- Locales: `ru` (default, unprefixed URLs), `es`, `en` (optional `/es/`, `/en/` prefix)
- Dictionaries: `src/lib/i18n/dictionaries/{ru,es,en}.ts`
- Middleware rewrites prefixed paths to existing Russian route tree
- Cookie `argentina-travel-locale` as fallback when no URL prefix

## Out of scope — E39.2 (future)

**CMS and editorial content translation** is **not** part of E39:

- Tour descriptions, blog posts, guide articles, immigration pages
- Organizer-authored fields in the tour editor
- Legal document body text (only link labels are translated in footer)
- SEO titles/descriptions for content pages beyond homepage and tours catalog stub

E39.2 will require:

1. Locale fields or parallel records in CMS / Supabase content tables
2. Locale-aware content routes or slug mapping per language
3. Editorial workflow for translators
4. hreflang for content URLs with translated slugs

Until E39.2, non-Russian visitors see **Russian editorial content** with translated UI chrome.

## E43: CMS i18n (E39.2 pilot)

Implemented locale-aware CMS documents for editorial content in admin and public resolvers.

### Scope

- Locales: `ru` (default), `es`, `en` — same as E39 interface chrome
- Document id format: `{docType}:{slug}:{locale}` (e.g. `blog:best-time-to-visit-argentina:en`)
- `content_documents.locale` column stores locale per row

### Admin

- **ContentDocumentEditorView**: locale tabs (RU / ES / EN) — switch between existing translations or create a new locale variant (`importFromSource` from TS)
- **Content inventory**: RU/ES/EN badges per document (green = published, amber = draft, gray = missing)
- **API** `GET /api/admin/content/documents?grouped=true` — list grouped by `docType + slug` with per-locale status

### Public resolution

- Server reads locale from cookie `argentina-travel-locale` (set by middleware on `/es/`, `/en/` prefix)
- `resolveWithPublishedCmsOverride` fallback chain: **requested locale → ru → TS file**
- Applied to: legal, blog, guide, destination, place pages and catalogs

### Pilot content

- Legal: `privacy` — es/en variants in `src/lib/cms/cms-i18n-pilot.ts`
- Blog: `best-time-to-visit-argentina` — es/en variants
- Seed via bulk import: `POST /api/admin/content/documents/bulk-import` with `{ "includeI18nPilot": true }`

### Out of scope (future)

- Tour descriptions and organizer fields
- Per-locale URL slugs and hreflang for content URLs
- Immigration / guide pillar TS-only pages

## Developer notes

- Use `t("key")` from `useLocaleCurrency()` in client components
- Use `loadMessages(locale)` + `t()` on server when needed
- `labelKey` on nav links resolves via `resolveNavLabel(link, t)`
- `addLocalePrefix(path, locale)` / `stripLocalePrefix(path)` for locale-aware links
- `buildHreflangAlternates(path)` for metadata alternates on public pages
