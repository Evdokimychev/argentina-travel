# E102 — Совместные поездки / групповой набор

## Что добавлено

- Таблицы Supabase:
  - `group_trip_listings` — набор группы к дате тура (`tour_id`, `slot_date`, `min/max_participants`, `status`, связь с `tour_availability_slots`).
  - `group_trip_members` — участники набора (`interested` | `confirmed` | `declined`).
- RLS: публичное чтение активных наборов, создание/участие — только авторизованным пользователям; полный доступ через `service_role`.
- API:
  - `GET/POST /api/group-trips` — список и создание набора.
  - `POST /api/group-trips/[id]/join|leave` — вступление и выход.
  - `PATCH /api/organizer/group-trips/[id]` — подтверждение или отмена организатором.
- Серверная логика: `src/lib/group-trips-server.ts`
  - проверка вместимости против слота E96 (`tour_availability_slots`);
  - автостатус `full` при достижении `max_participants`;
  - уведомления при достижении `min_participants` (`notifications-server`).
- UI:
  - блок «Совместная поездка» на странице тура;
  - `/profile/group-trips` — мои наборы;
  - `/organizer/group-trips` — управление для организатора.

## Поток туриста

1. На странице тура выбирает дату, минимальный и максимальный состав, создаёт набор.
2. Другие туристы присоединяются через «Присоединиться».
3. При достижении минимума — in-app (+ email организатору) уведомление.
4. Организатор подтверждает состав; участники получают статус `confirmed`.
5. Бронирование и оплата — отдельным этапом (E102 не реализует оплату и split payment).

## Поток организатора

1. Раздел «Наборы группы» в кабинете.
2. Видит состав, дату, прогресс `N / min / max`.
3. «Подтвердить состав» доступно при `memberCount >= min_participants`.
4. «Отменить набор» закрывает набор (`cancelled`).

## Интеграция с E96

- При создании набора `max_participants` ограничивается `availableCount` слота.
- Закрытые и sold-out слоты не принимают новые наборы.
- Поле `availability_slot_id` сохраняет связь со слотом, если он есть в БД.

## Статусы набора

- `open` — набор открыт.
- `full` — достигнут максимум участников.
- `confirmed` — организатор подтвердил состав.
- `cancelled` — набор отменён.

## Примечание по правам

- Запись через API выполняется с `service_role` там, где нужна атомарная бизнес-логика.
- RLS покрывает прямой доступ клиента Supabase.
- `npm run build` и `npm run rls-audit` должны проходить после миграции.
