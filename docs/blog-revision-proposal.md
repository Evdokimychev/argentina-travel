## 2. Сильные стороны

1. **Двухклассовая модель с SEO-защитой.** Класс B получает `noIndex: true`; в `generateMetadata` — `robots: { index: false }`. Это правильное решение для 222 шаблонных статей.

2. **Единый автор.** `BLOG_EDITORIAL` («Редакция «Пора в Аргентину»») — соответствует E-E-A-T; фейковые персонажи убраны.

3. **Rich-статьи — эталон качества.** Структура блоков (stats, seasons, spots, callouts, FAQ), литературный русский, фактические оговорки («актуально на июнь 2026»), FAQ JSON-LD. TS синхронизирован с MD (lede, intro, FAQ совпадают с `docs/Национальный-парк-Игуасу.md`).

4. **Ручные секционные статьи.** Единый каркас «Кратко → Практика → Бюджет → FAQ», `relatedResources`, tour embeds на `patagonia-packing-list`, `buenos-aires-neighborhoods`, `mendoza-wine-route`.

5. **UX чтения.** `ContentReadingLayout` + TOC (из секций или rich), `SharePageLinkButton`, боковая панель с хаб-ссылками (`BLOG_HUB_LINKS`).

6. **Карусель галереи.** Доступность: `role="region"`, стрелки, lightbox, Escape.

7. **Инфраструктура изображений.** Image-provider, manifest, локальные hero для ручных slug, gallery slots для rich.

8. **Патагония overrides.** 28 полноценных статей в `patagonia.ts` — образец для масштабирования editorial на другие категории.

9. **Фактическая правка стейка.** В `argentinian-steak-guide` отрубы исправлены: bife de chorizo = стриплойн, ojo de bife = рибай.

### 6.2. Синхронизация MD → TS

**Проблема:** rich-статьи живут в TS (~650 строк); MD в `docs/` — authoring source, но без автоматической сборки.

**Предложение:**

1. Скрипт `scripts/sync-rich-articles.mjs`: парсит MD → генерирует/валидирует TS (или JSON intermediate).
2. CI check: hash MD vs hash embedded `updatedLabel` / checksum в frontmatter.
3. Плейсхолдеры MD → typed blocks: `<!-- map: lat,lng -->` → `{ type: "map", lat, lng }`.
