# Источники данных интерактивной карты

Страница: `/mapa-argentina`. Сборка объектов: `src/lib/map-objects-server.ts` → `fetchMapObjects()`.

## Откуда берутся объекты

| Тип метки | Источник | Связанные страницы |
|---|---|---|
| Города, нацпарки, достопримечательности | `src/data/places-seed.ts` (или Prisma при `PLACES_USE_DB=true`) | `/places/[slug]`, подборки, маршруты |
| Экскурсии (tour) | `fetchMarketplaceTours()` | `/tours/[slug]` |
| Аэропорты | `src/data/argentina-airports.ts` | `/guide/kak-dobratsya` |
| Рейсы между аэропортами | `src/data/argentina-flight-routes.ts` | карточка аэропорта на карте |
| Транспортные узлы | `src/data/argentina-transport-hubs.ts` | places / destinations |
| Маршруты туров | `src/data/tour-routes.ts` | `/tours/[slug]` |
| Границы регионов | `src/data/argentina-regions.ts` | `/destinations/*` |

## Правила при добавлении мест

1. Новые города/достопримечательности добавлять в `PLACES_SEED` — так место автоматически
   получает страницу `/places/[slug]`, попадает на карту, в поиск и связи (relations).
2. `region` — строго из таксономии: «Центр и Пампа», «Патагония», «Северо-запад»,
   «Северо-восток», «Куйо», «Огненная Земля».
3. Факты (высоты, статусы ЮНЕСКО, сезоны) проверять; непроверяемое — смягчать
   («уточняйте перед поездкой»), см. `editorial-standard.mdc`.
4. Фото подключаются через `media-library/manifest.json` (`placeId` = slug места).
   Без фото карточка на карте показывает градиент по цвету категории — это нормально.
5. **Каждое место — только свои фото.** Заимствование обложек соседних мест запрещено
   (`PLACE_COVER_FALLBACK_MAP` держим пустым). Новые фото загружаются скриптом
   `scripts/fetch-place-media.mjs` (Wikimedia Commons, точечные запросы по месту);
   он же убирает дубликаты между местами.

## Будущая база знаний (первоисточник)

Готовится внешняя база знаний сайта (владелец собирает её отдельно). Когда она появится:

- База знаний станет **первоисточником** описаний мест, городов и маршрутов;
  `PLACES_SEED` и описания в `argentina-airports.ts` нужно будет сверить/перегенерировать из неё.
- Точка интеграции — те же модули данных: структура `SeedPlace` уже покрывает
  нужные поля (описания, сезоны, теги, координаты). Идентификатор для сопоставления — `slug`.
- Не хардкодить факты в компонентах UI: весь контент держать в `src/data/*`,
  чтобы замена источника не требовала правок интерфейса.
