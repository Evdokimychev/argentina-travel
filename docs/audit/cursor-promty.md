# Промпты для Cursor — внедрение правок по аудиту goargentina.ru

Этот файл — пошаговая инструкция и готовые промпты. Копируй блоки по очереди.

---

## Шаг 0. Как дать Cursor доступ к отчётам

**Вариант A (рекомендуется).** Скопируй все файлы `audit-*.md` в репозиторий сайта, в папку `docs/audit/`. Тогда в чате Cursor можно ссылаться на них через `@`:
- `@docs/audit/audit-goargentina-etap-1.md`
- `@docs/audit/audit-etap-2-razdel-1-glavnaya.md` и т.д.

**Вариант B.** Если не хочешь класть файлы в репозиторий — открой нужный `.md`, скопируй его текст и вставь прямо в промпт (вместо `@...` ссылки).

**Модель в Cursor:** для массовых правок — **Claude 4.6 Sonnet** (Agent), для сложных (каталог/Tripster, цены, гражданство) — **Claude Opus 4.8**.

**Режим:** Agent (он сам находит и правит файлы). Включи показ диффа.

## Шаг 1. Стартовый промпт (задаёт правила — отправь ОДИН раз в начале сессии)

```
Ты работаешь над сайтом «Пора в Аргентину» (Next.js, App Router, i18n, хостинг Vercel).
У меня есть отчёты внешнего контент-аудита в папке docs/audit/. Мы будем внедрять правки по ним.

Жёсткие правила на всю сессию:
1. Работай маленькими партиями: одна задача (по её ID, например G-01) за раз.
2. Ничего не выдумывай. Если данных нет — оставь TODO-комментарий и спроси меня, не придумывай цифры, цены, отзывы, даты.
3. Не удаляй контент и не меняй структуру без моего подтверждения. Сначала покажи план.
4. После каждой партии: покажи список изменённых файлов и diff, затем прогоняй `npm run build` (или `next build`) и `npm run lint`. Если ломается — чини до зелёного.
5. Не трогай чужой контент (импорт Tripster) — по нему отдельное решение.
6. Тексты — на русском, в стиле качественного турпортала (Lonely Planet / Visit Argentina): без «воды» и канцелярита; иностранные термины — в скобках при первом упоминании (например, «Перито-Морено (Perito Moreno)»).

Подтверди, что понял правила, и жди мою первую задачу.
```
