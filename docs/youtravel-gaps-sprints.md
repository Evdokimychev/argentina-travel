## Спринт 6 — Каталог и discovery (quick wins)

**Цель:** показать на карточках то, что API уже отдаёт.

- [x] **6.1** Бейджи «Мгновенная бронь» / «Гарантия проведения» на карточке каталога (`instant_booking`, `guarantee`, serp)
- [x] **6.2** Тематические метки из `types[]` / `tags[]` / `main_type` (не только «Партнёр YouTravel.me»)
- [x] **6.3** Скидка заезда: `priceDiscountValue` → зачёркнутая цена в блоке дат
- [x] **6.4** Проброс `partnerInstantBooking` / `partnerTourGuaranteed` / `partnerThematicTags` в `TourListing`

**Критерий:** карточка youtravel-тура в `/tours` показывает бейджи и теги при наличии данных в payload.

## Спринт 7 — Карточка тура (UX)

**Цель:** детальная страница ближе к YouTravel и Tripster.

- [x] **7.1** Локации дня в программе (`days[].locations_data[].name`)
- [x] **7.2** Блок «Условия бронирования» (предоплата, мгновенная бронь, отмена) — как у экскурсий
- [x] **7.3** ~~Партнёрская ссылка заезда (`offer_id` в affiliate)~~ — уже есть в `PartnerTourBookingContactSection`
- [x] **7.4** ~~Ссылка на профиль эксперта~~ — уже есть в `PartnerTourOrganizerSection` (`guide.url`)

**Критерий:** в программе видны точки маршрута по дням; под датами — условия брони единым блоком.

## Спринт 9 — Live booking (после активации API)

- [x] **9.1** Проверка `POST /v1/booking-requests` / `/v1/orders` в prod
- [x] **9.2** Статусы заявки в ЛК туриста
- [x] **9.3** Webhook / polling статуса (если появится в API)

**Блокер:** активация API в кабинете YouTravel (см. спринт 1). Код готов — после активации заработают live-заказы, refresh в ЛК и cron/webhook синхронизация.

**Подготовка без API (можно до разблокировки):** улучшение ЛК, verify-скрипт, polling-каркас.

## Спринт 10 — Метаданные каталога (средний объём)

**Цель:** фильтры и карточки используют реальные данные API, а не hardcode.

- [x] **10.1** `rowToListing`: comfort / difficulty / minimumAge / childrenAllowed из payload (`comfort_data`, `activity_data`, `age_from`)
- [x] **10.2** `comfortHtml` в `buildYouTravelPartnerContent` → блок «Комфорт» на карточке
- [x] **10.3** Фильтры «Комфорт» и «Нагрузка» применяются к YouTravel-турам (сейчас partner skip)
- [x] **10.4** Unit-тесты маппинга listing + filter-tours

**Критерий:** фильтр «Комфорт: Премиум» сужает youtravel-туры; на детальной — секция комфорта при наличии описания.

## Спринт 11 — Эксперт и бронирование (средний объём)

**Цель:** discovery эксперта и готовность к live API.

- [x] **11.1** Фильтр `?organizer=youtravel-expert-{id}` — проверка, chip в каталоге, ссылка с карточки тура
- [x] **11.2** ЛК: статусы YouTravel-заявок — ссылка на партнёра, дата обновления, refresh
- [x] **11.3** `youtravel:verify` — probe `POST /v1/booking-requests` (dry-run / expect 401)
- [x] **11.4** Каркас polling статуса (no-op без API)

**Критерий:** из карточки тура — «Все туры эксперта»; в ЛК понятный статус affiliate_fallback vs api.
